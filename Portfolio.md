## 🧠 STAR Method Reflections

### 1. Reflection on Coding my CV in Markdown

**Situation**  
I needed to present my professional CV inside a GitHub portfolio using only Markdown syntax, without word processors.

**Task**  
Convert my traditional PDF CV into clean, readable Markdown while maintaining structure, headings, lists, and links.

**Action**  
- Used GitHub Flavoured Markdown (GFM)  
- Created headings (`#`, `##`, `###`) for sections  
- Used bullet lists for skills and projects  
- Added inline links for email, phone, and LinkedIn  
- Tested rendering on GitHub and locally

**Result**  
A fully functional, version-controlled CV that is easy to update, share, and display on GitHub Pages. I learned that Markdown is powerful for technical documentation and portfolio building.

### 3. Reflection on Using GitHub Pages

**Situation**  
I had to deploy my entire digital portfolio to a live website using GitHub Pages.

**Task**  
Configure a GitHub repository to serve Markdown files as a website, accessible via a public URL.

**Action**  
- Created a new repository named `PRP370-digital-portfolio`  
- Enabled GitHub Pages in repository Settings → Pages → Branch `main` → `/root`  
- Used `index.md` as the entry file  
- Committed and pushed all files  
- Waited 1–2 minutes for deployment

**Result**  
My portfolio is live at `https://[username].github.io/PRP370-digital-portfolio/`. I learned how static site hosting works, the importance of relative links, and why GitHub Pages is ideal for student portfolios.
